The user cannot leave immediately.

After successful eSewa payment, the user can leave. Their old transactions remain visible as Former Member for transparency.
Case B: Others owe the user money
Example: “Ritika and Suman owe you Rs 1,200.”
The user can leave, but the receivable remains active.
The app shows:
You are still owed Rs 1,200
“You can leave the group now. Your pending receivables will remain visible, and members can still pay you.”
Actions:

Send Reminder First
Cancel

User is removed from future expenses.
Their past expenses remain visible.
People who owe them still see payment prompts.
The user can still receive settlement payments.
Case C: User has zero balance
The user can leave immediately.
Message:

“You have no pending balance. Past records will remain visible to group members.”
Actions:
Leave Group
Cancel
Case D: Group admin wants to leave
If the user is admin, they must first assign a new admin.
Flow:
Choose New Admin → Confirm Transfer → Leave Group
The group cannot be left without an active admin.

Savings Circle group
Savings Circle is stricter than a normal expense group because it is a rotating savings commitment.
Case A: Savings Circle has not started
A member can leave or decline before all members accept.
Result:
Member is removed.
Contribution amount or payout schedule is recalculated.
Remaining members must review the updated schedule.
Case B: Current contribution is unpaid
The member cannot leave until the current due contribution is resolved.
Message:
Contribution pending
“You have Rs 5,000 due for Cycle 3. Please contribute or ask the admin to cancel your participation before leaving.”
Actions:
Pay Contribution
Request Admin Review
Cancel
Case C: User already received Savings Circle payout
The user cannot leave casually because they have already received the community pot.
Message:
Payout already received
“You received Rs 25,000 in Cycle 2. To leave, you must complete your remaining contributions or settle them in advance.”
Actions:
Pay Remaining Contributions
Request Exit Approval
View Agreement
This prevents a user from taking the payout and leaving without completing future contributions.
Case D: User has contributed but not yet received payout
The user can request exit, but it requires approval.
Flow:
Request Exit → Admin Review → Members Reconfirm Schedule
The app shows:
Contributions already paid
Whether refund is applicable according to group agreement
Updated payout order
Updated pot amount
Updated service fee
Savings Circle exit is not instant because changing one member affects everyone’s payout schedule.

Ritika
Suman
The item is split only among those 3 people:
Aayush: Rs 100
Ritika: Rs 100
Suman: Rs 100
Others: Rs 0 for that item
If the split is not equal, the user can choose Split by Shares.
Example:
Chicken Momo — Rs 300
Aayush: 2 shares
Ritika: 1 share
Suman: 1 share
Result:
Aayush: Rs 150
Ritika: Rs 75
Suman: Rs 75
The item row shows small avatar bubbles so everyone can quickly see who was assigned to that item.
Service charge, VAT, and tips are allocated proportionally based on each person’s item total by default. The user can change this to equal allocation if needed.

3. Equal split rounding: Rs 100 split among 3 people
The app should never show:
Rs 33.33 + Rs 33.33 + Rs 33.33 = Rs 99.99
That creates a missing paisa and breaks trust.
So the app uses this rule:
Always preserve the exact total.
For Rs 100 split among 3 people:
Person 1: Rs 33.34
Person 2: Rs 33.33
Person 3: Rs 33.33
Total: Rs 100.00
Who gets Rs 33.34?
Default rule:
The extra Rs 0.01 goes to the payer’s own share if the payer is included in the split.
Example:
Aayush paid Rs 100 for Aayush, Ritika, and Suman.
Split result:
Aayush: Rs 33.34
Ritika owes Aayush: Rs 33.33
Suman owes Aayush: Rs 33.33
Why this works:
The payer absorbs the tiny rounding difference.
Other members are not randomly charged extra.
The total remains mathematically correct.
If the payer is not part of the split, the extra Rs 0.01 goes to the first selected member by group order.
If there are multiple payers, the extra Rs 0.01 goes to the payer who paid the largest amount.
The confirmation screen shows:
Rounding adjustment: +Rs 0.01 applied to Aayush
Users can tap this row and manually move the rounding adjustment to another member if they want.

4. Multiple people paid for one group expense
The app supports multiple payers inside Add Expense.
Example:
Total bill: Rs 6,000 Members: 6 Split: Equal Each person’s share: Rs 1,000
Paid by:
Rajan paid Rs 4,000
Ritika paid Rs 2,000
Calculation:

The app then creates optimized settlement lines:
Suman → Rajan: Rs 1,000
Anjali → Rajan: Rs 1,000
Nabin → Rajan: Rs 1,000
Bikash → Ritika: Rs 1,000
The formula is simple:
Net balance = Amount paid − Personal share
Positive balance means the person should receive money. Negative balance means the person owes money.
The app pairs people who owe with people who should receive, then creates the smallest clear settlement list.

Scan receipts using OCR

Updated split engine rules
Rule 1: Every split must equal the original total
The app never allows a split total mismatch.
If the bill is Rs 100, the assigned total must be exactly Rs 100.
Rule 2: Equal split uses precise rounding
For Rs 100 among 3:
Rs 33.34
Rs 33.33
Rs 33.33
The rounding adjustment is shown clearly.
Rule 3: Item split supports partial participation
An item can be assigned to:
One member
Two members
Any subset of group members
All members
Rule 4: Item split supports shares
Useful for cases like:
One person ate more
Two people shared most of an item
Kids/adults split differently
Someone only had drinks
Rule 5: Multiple payers are supported
Each expense can have one or more payers.
The app calculates:
Paid − Share = Net Balance
Rule 6: Group balances are netted
The app does not show unnecessary payment loops.
Instead of:
A owes B
B owes C
C owes A
It simplifies the final settlement wherever possible.
Rule 7: Past records remain visible
Leaving a group or removing a connection never deletes financial history.
Rule 8: Savings Circle commitments are stronger than expense groups
A user who has received payout cannot leave without settling remaining obligations.
